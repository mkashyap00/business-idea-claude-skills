# Phase 1 Synthesis & Context Engineering Checkpoint

Your goal is to act as a strict sequential bottleneck between the foundational strategy phase and the testing phase. You must prevent "contextual drift" by taking the massive amount of data generated in Phase 1, engineering it into a hyper-focused markdown document, and forcing an executive "taste decision" before allowing the pipeline to proceed. 

## Anti-Rationalization Table
Do not skip steps. Large language models frequently rationalize skipping verification steps. If you are tempted to take a shortcut, refer to these strict rebuttals:
*   **Excuse:** "The transition to Phase 2 is obvious based on the BMC; I will automatically pass the context to the `/initial-wedge` agent." -> **Rebuttal:** "No. You must halt the pipeline and use the `AskUserQuestion` tool to force an executive decision. Execution without approval is forbidden."
*   **Excuse:** "I will just summarize the data in the chat window." -> **Rebuttal:** "No. You must use the `Write` tool to create a concrete markdown file so that the Phase 2 agents have a persistent document to read from."

## Procedural Workflow

### Step 1: Digest Phase 1 Outputs
Gather and analyze the raw strategic data generated by the three preceding frameworks:
1.  **First Principles Audit:** The core "atomic truths" and irreducible facts of the business problem and "Linchpin Assumption".
2.  **Blue Ocean Strategy:** The differentiated value curve established by the ERRC (Eliminate, Reduce, Raise, Create) framework.
3.  **Business Model Canvas:** The AI-Enhanced 9 building blocks mapping out value creation and delivery.

### Step 2: Context Engineering & File Generation
Strip away the conversational bloat and framework-specific formatting. Synthesize the digested outputs into a single, highly potent, and focused strategic narrative that clearly defines *what* the business is, *how* it creates uncontested value, and the *core mechanics* of its operation. 

Use the `Write` tool to save this narrative into a standalone file named `phase1_engineered_context.md` in the current directory.

### Step 3: Executive Decision Checkpoint (AskUserQuestion)
You must NOT automatically proceed to Phase 2. Once the file is written, use the `AskUserQuestion` tool to present your engineered context to the user and force an executive decision.

Structure your prompt strictly as follows:
*   **Re-ground:** State that `phase1_engineered_context.md` has been successfully generated.
*   **Simplify:** Highlight the strongest strategic advantages and the most obvious operational bottlenecks found in the document.
*   **Recommend:** Propose whether the current positioning is strong enough to move into designing tests, or if a structural flaw requires revisiting a previous framework.
*   **Options:** Give the user clear multiple-choice options to lock in their decision:
    *   *A: Lock in this business model and proceed to Phase 2 (Wedge Design).*
    *   *B: Revise the Blue Ocean positioning or BMC blocks.*
    *   *C: Halt the project.*
