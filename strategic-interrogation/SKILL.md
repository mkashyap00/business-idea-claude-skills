# Strategic Interrogation Checkpoint

Your goal is to act as the final strategic safeguard by digesting the concrete critiques from the parallel review sprints, forcing a clear executive decision, and generating the formalized hand-off documents for Phase 4 (Test Deployment).

## Anti-Rationalization Table
Do not skip steps. If you are tempted to take a shortcut, refer to these strict rebuttals:
*   **Excuse:** "I already know what the subagents found from the conversation history." -> **Rebuttal:** "No. You must use the `Read` tool to explicitly ingest all four critique `.md` files to prevent context degradation."
*   **Excuse:** "I will just provide the summary in the chat and move to the next phase." -> **Rebuttal:** "No. You must use the `AskUserQuestion` tool to halt the pipeline and force an executive Go/No-Go decision."
*   **Excuse:** "I'll output a text summary instead of the HTML and Markdown files." -> **Rebuttal:** "No. The concrete artifacts are mandatory. You must use the `Write` tool to create both the standalone HTML dashboard and the intermediate Markdown file for Phase 4."

## Procedural Workflow

### Step 1: Ingest the Review Sprint Artifacts
You must use the `Read` tool to sequentially ingest the concrete evidence generated by the Phase 3 subagents. Read the following files from the current directory:
1.  `investor_critique.md`
2.  `customer_critique.md`
3.  `realist_autopsy.md`
4.  `macro_risks.md`

### Step 2: The 5-Domain Factual Check
Evaluate and synthesize the critiques from the four files into a single executive summary. This synthesis must catch any remaining blind spots across these five domains:
1.  **Investigative:** What are the hard, proven facts vs. our untested assumptions?
2.  **Speculative:** What are the most dangerous "what-ifs" identified by the realist red team and macro risk analysis?
3.  **Productive:** What is the actual operational effort required to build this initial wedge?
4.  **Interpretive:** How are we defining the metrics for success or failure based on the investor and customer reviews?
5.  **Subjective:** What biases or inconvenient truths are we currently ignoring?

### Step 3: Executive Decision Checkpoint (AskUserQuestion)
You must NOT automatically proceed. Use the `AskUserQuestion` tool to present the final 5-domain synthesis to the user and force an executive Go/No-Go decision.

Structure your `AskUserQuestion` prompt strictly as follows:
*   **Re-ground:** Briefly state that the Phase 3 Review Sprints are complete and the four critique files have been fully ingested.
*   **Simplify:** Present the 5-Domain Executive Summary. Highlight the major tradeoffs, bottlenecks, or fatal flaws discovered in plain English.
*   **Recommend:** Propose the strongest, narrowest path forward based on the data.
*   **Options:** Give the user clear multiple-choice options to lock in their decision:
    *   *A: GO - Lock in the plan and generate Phase 4 hand-off files.*
    *   *B: NO-GO - Pivot the core idea or wedge based on the vulnerabilities.*
    *   *C: HALT - Terminate the project.*

### Step 4: Generate the Hand-off Artifacts (Write)
If the user selects "Option A: GO", you must use the `Write` tool to create TWO standalone, dependency-free files in the current directory:
1.  `approved_strategy_synthesis.md`: A structured markdown document containing the finalized 5-domain synthesis and the approved go-to-market parameters. This serves as the unbreakable data contract that the Phase 4 agents (`/lanchester-go-to-market` and `/mvp-test-strategist`) will read.
2.  `executive_decision_dashboard.html`: A visual, dependency-free HTML file utilizing CSS Grid/Flexbox to display an interactive "Executive Summary Dashboard". Include a visual radar chart or a 5-pillar grid representing the 5 domains (Investigative, Speculative, Productive, Interpretive, Subjective), color-coding areas of high risk (red) vs. validated strength (green). 

Inform the user when both files have been successfully generated and that Phase 4 is ready to begin.
